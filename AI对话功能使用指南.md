# AI对话功能使用指南

## 功能概述

AI对话助手是刷题管理系统的核心功能之一，它能够：

1. **分析已掌握知识点**：根据刷题数量和难度，智能判断你掌握了哪些知识点
2. **识别薄弱环节**：找出刷题较少的知识点，制定强化计划
3. **智能推荐题目**：基于已掌握知识点，推荐"跳一跳够得着"的进阶题目
4. **提供学习建议**：给出具体的训练方向和学习路径

## 使用方法

### 1. 启动Web服务

```bash
cd 刷题管理系统
python web/app.py
```

### 2. 访问主页

打开浏览器访问：http://localhost:5000

### 3. 开启AI对话

点击页面右上角的 **"💬 开启AI对话助手"** 按钮

### 4. 与AI对话

在对话框中输入问题，例如：

- "分析一下我现在的水平"
- "我在动态规划方面比较弱，推荐几道题目"
- "基于我已经掌握的知识点，给我推荐一些进阶题目"
- "我想提升图论的水平，有什么建议？"
- "我应该重点攻克哪些知识点？"

## AI的工作原理

### 1. 数据收集

AI会自动收集以下数据：

- **总题数**：本地题目 + 平台题目的总数
- **知识点分布**：各知识点的刷题数量
- **难度分布**：各难度段的刷题数量
- **已做题目列表**：避免重复推荐
- **可用题库**：数据库中存储的平台题目

### 2. 知识点掌握度判断

| 刷题数量 | 掌握度 | 说明 |
|----------|--------|------|
| 0题 | 未接触 | 知识盲区 |
| 1-2题 | 初步接触 | 需要基础练习 |
| 3-7题 | 初步掌握 | 有一定基础 |
| 8题以上 | 熟练掌握 | 可以进行进阶训练 |

### 3. 推荐策略

#### 基于已掌握知识点的进阶路径

对于已掌握的知识点，AI会推荐：

- **深度扩展**：该知识点的高难度变种
- **交叉融合**：与其他知识点结合的综合题
- **实际应用**：该知识点在竞赛中的经典题型

示例：
- 熟练掌握"基础DP" → 推荐"DP优化"、"斜率优化"
- 熟练掌握"最短路" → 推荐"差分约束"、"网络流"

#### 薄弱知识点的强化

- 从简单题开始建立信心
- 推荐2-3道同一知识点的梯度练习
- 提供学习建议和常见套路

#### 知识盲区的开拓

- 优先选择与已掌握知识点相关的新知识
- 推荐入门教程或经典入门题

### 4. 难度匹配

AI会分析：

- **主力难度**：你刷得最多的难度段
- **平均难度**：所有题目的平均难度
- **难度跨度**：题目分布的广度

推荐的题目难度通常比你的主力难度高 **1-2级**。

## 配置要求

### 1. DeepSeek API Key

AI对话功能需要 DeepSeek API Key。

#### 获取方式

1. 访问 https://platform.deepseek.com/
2. 注册并登录
3. 在"API密钥"页面创建新密钥

#### 配置方法

编辑 `config.json` 文件：

```json
{
  "deepseek_api_key": "你的API密钥"
}
```

或者设置环境变量：

```bash
set DEEPSEEK_API_KEY=你的API密钥
```

### 2. 可用题库（重要）

AI推荐题目需要依赖数据库中存储的平台题目。

#### 如何获取题库

1. 访问 **设置页面**（http://localhost:5000/settings）
2. 配置平台账号（洛谷、Codeforces等）
3. 点击"爬取平台题目"按钮
4. 等待爬取完成

#### 注意事项

- 题库数据存储在 `data/competition.db` 数据库中
- 只能推荐已爬取到数据库中的题目
- 建议定期更新题库（每周一次）

## 常见问题

### Q: AI推荐的都是我做过的题目？

A: 可能原因：
- 数据库中没有足够的平台题目
- 建议先爬取更多题库数据

### Q: AI回答很慢或超时？

A: 可能原因：
- API调用频率限制（错误码14003）
- 解决：等待1-2分钟后重试

### Q: AI推荐的题目太难了？

A: 可能原因：
- 你的主力难度较低，但AI推荐的难度过高
- 解决：明确告诉AI"推荐简单一点的题目"

### Q: 如何获取更精准的推荐？

A: 建议：
1. 确保数据库中有足够多的平台题目
2. 明确说明你的需求，如"我想提升图论能力"
3. 提供更多上下文，如"我最近在学DP"

## 高级技巧

### 1. 多轮对话

AI支持上下文记忆，可以进行多轮对话：

```
你: 分析一下我现在的水平
AI: [水平分析]
你: 针对我的薄弱点，推荐几道题目
AI: [题目推荐]
你: 第一道题有点难，有没有简单点的？
AI: [更简单的题目]
```

### 2. 明确指定知识点

```
你: 我想专门提升动态规划
你: 推荐几道斜率优化的题目
你: 我需要练习数论中的中国剩余定理
```

### 3. 指定难度

```
你: 推荐几道难度4的图论题
你: 我现在水平是难度3，推荐比这稍微难一点的
```

## 技术细节

### API端点

- **路径**: `/api/chat`
- **方法**: POST
- **请求体**:
```json
{
  "message": "用户消息",
  "history": [
    {"role": "user", "content": "上一轮用户消息"},
    {"role": "assistant", "content": "上一轮AI回复"}
  ]
}
```

### 上下文数据

AI会自动获取以下数据：

```python
{
  "total": "总题数",
  "solved": "已解决数",
  "by_category": {"知识点": 数量},
  "by_difficulty": {难度: 数量},
  "by_platform": {"平台": 数量},
  "solved_problems": [
    {"title": "题目名", "file_name": "文件名"}
  ],
  "available_problems": [
    {
      "title": "题目名",
      "platform": "平台",
      "problem_id": "题目ID",
      "difficulty": 难度,
      "tags": ["知识点1", "知识点2"]
    }
  ]
}
```
