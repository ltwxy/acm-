# 智能刷题管理系统设计文档

## 🎯 系统目标

自动识别、分类、统计本地刷题文件，并抓取在线平台的题目信息，生成可视化的学习进度报告。

## 🗄️ 数据库设计

### problems 表 - 题目信息

```sql
CREATE TABLE problems (
    id              INTEGER PRIMARY KEY AUTOINCREMENT,
    file_path       TEXT UNIQUE NOT NULL,      -- 文件完整路径
    file_name       TEXT NOT NULL,              -- 文件名
    title           TEXT,                       -- 题目标题
    platform        TEXT,                       -- 来源平台: luogu/cf/lc/unknown
    problem_id      TEXT,                       -- 平台题号
    difficulty      INTEGER,                    -- 难度: 1-10
    status          TEXT DEFAULT 'pending',     -- 状态: pending/solved/review
    
    -- 分类信息
    category        TEXT,                       -- 主分类
    subcategory     TEXT,                       -- 子分类
    tags            TEXT,                       -- 标签JSON数组
    
    -- 代码分析结果
    algorithms      TEXT,                       -- 检测到的算法JSON
    data_structures TEXT,                       -- 检测到的数据结构JSON
    code_features   TEXT,                       -- 代码特征JSON
    
    -- 统计信息
    lines_of_code   INTEGER,
    submit_count    INTEGER DEFAULT 0,
    
    -- 时间戳
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    solved_at       TIMESTAMP
);
```

### categories 表 - 分类定义

```sql
CREATE TABLE categories (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    name        TEXT UNIQUE NOT NULL,           -- 分类名
    parent_id   INTEGER,                        -- 父分类ID
    keywords    TEXT,                           -- 关键词JSON
    patterns    TEXT,                           -- 代码特征正则
    description TEXT,
    color       TEXT                            -- 统计图颜色
);
```

### daily_stats 表 - 每日统计

```sql
CREATE TABLE daily_stats (
    date            DATE PRIMARY KEY,
    problems_solved INTEGER DEFAULT 0,
    new_problems    INTEGER DEFAULT 0,
    by_category     TEXT,                       -- JSON
    by_difficulty   TEXT                        -- JSON
);
```

## 🔍 自动分类规则

### 1. 文件名解析规则

| 模式 | 平台 | 示例 |
|------|------|------|
| `P\d+` 或 `p\d+` | 洛谷 | `P1314聪明的质检员.cpp` |
| `CF\d+` | Codeforces | `CF1234A.cpp` |
| `\d+` | 牛客/其他 | `1.cpp`, `A.cpp` |
| `LC\d+` | LeetCode | `LC1两数之和.cpp` |

### 2. 代码特征识别

```python
CODE_PATTERNS = {
    # 动态规划
    'dp': {
        'keywords': ['dp', 'f[', 'dp[', '记忆化', '状态转移'],
        'patterns': [
            r'dp\[.*\].*=.*',           # dp[i] = 
            r'for.*dp.*max|min',         # 循环+最值
            r'vector.*dp',               # dp数组定义
        ]
    },
    
    # BFS
    'bfs': {
        'keywords': ['bfs', 'queue', '队列', '广度'],
        'patterns': [
            r'queue.*<.*>.*q',
            r'while.*!q\.empty',
            r'q\.push|q\.pop',
        ]
    },
    
    # DFS
    'dfs': {
        'keywords': ['dfs', '深度', '递归', '回溯'],
        'patterns': [
            r'void\s+dfs\s*\(',
            r'dfs\s*\([^)]+\)',          # dfs(...)调用
        ]
    },
    
    # 图论
    'graph': {
        'keywords': ['图', '边', '点', '邻接', 'vector<int>.*g'],
        'patterns': [
            r'struct.*edge|Edge',
            r'add_edge|addedge',
            r'vector.*g\[|vector.*adj',
        ]
    },
    
    # 并查集
    'union_find': {
        'keywords': ['并查集', 'fa[', 'parent', '连通'],
        'patterns': [
            r'find\s*\(.*\)|int\s+find',
            r'union|merge.*set',
            r'fa\[.*\]=fa\[',
        ]
    },
    
    # 线段树
    'segment_tree': {
        'keywords': ['线段树', 'seg', 'tree', 'build'],
        'patterns': [
            r'struct.*SegTree|SegmentTree',
            r'build\s*\(.*l.*r',
            r'pushup|pushdown',
        ]
    },
    
    # 树状数组
    'bit': {
        'keywords': ['树状数组', 'BIT', 'lowbit'],
        'patterns': [
            r'lowbit\s*\(',
            r'c\[.*\].*\+=',
            r'int.*lowbit',
        ]
    },
    
    # 二分
    'binary_search': {
        'keywords': ['二分', 'mid', 'lower_bound', 'upper_bound'],
        'patterns': [
            r'while.*l.*<.*r',
            r'mid.*=.*l.*\+.*r',
            r'lower_bound|upper_bound',
        ]
    },
    
    # 贪心
    'greedy': {
        'keywords': ['贪心', '排序', 'sort'],
        'patterns': [
            r'sort.*begin.*end',
            r'贪心',
        ]
    },
    
    # 数学
    'math': {
        'keywords': ['gcd', 'lcm', '质数', '素数', 'mod', '快速幂'],
        'patterns': [
            r'gcd\s*\(|lcm\s*\(',
            r'pow_mod|qpow|quick_pow',
            r'%\s*MOD|%\s*mod',
        ]
    },
}
```

### 3. 分类优先级

```
高优先级（直接判定）：
- 文件名包含明确算法名：如 "bfs", "dp", "dijkstra"
- 代码中出现特定模板结构

中优先级（加权计算）：
- 关键词匹配数量
- 代码特征匹配数量

低优先级（辅助判断）：
- 文件大小（模板题通常较短）
- 注释内容分析
```

### API接口封装

```python
class PlatformAPI:
    """平台API基类"""
    
    async def fetch_problem(self, problem_id: str) -> dict:
        """
        获取题目信息
        Returns: {
            'title': '题目名称',
            'difficulty': 难度值,
            'tags': ['标签1', '标签2'],
            'url': '题目链接'
        }
        """
        pass

class LuoguAPI(PlatformAPI):
    BASE_URL = 'https://www.luogu.com.cn'
    
class CodeforcesAPI(PlatformAPI):
    BASE_URL = 'https://codeforces.com/api'
    
class LeetCodeAPI(PlatformAPI):
    BASE_URL = 'https://leetcode.cn/graphql'
```
